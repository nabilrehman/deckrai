**Presentation Type:** Technical deep-dive

**Audience:** Technical architects and data engineers

**Goal:** Educate on GCP data platform architecture and implementation details

**Content:**
We implemented a data lakehouse on BigQuery with the following architecture:

1. **Ingestion Layer (Bronze)**
   - Using CData Sync for real-time data ingestion from Salesforce and Pendo
   - Raw data stored in Bronze dataset with minimal transformation
   - Schema evolution handled programmatically via metadata tables

2. **Transformation Layer (Silver/Gold)**
   - DBT/Dataform for transformations
   - Domain-driven Silver layer organization
   - Business-ready Gold layer for analytics

3. **Governance & Security**
   - DataPlex for data taxonomy and discovery
   - Column-level security with policy tags
   - RBAC using Google Groups synchronized with Microsoft AD
   - Centralized custom roles at organization level

4. **AI & Analytics Integration**
   - BigQuery ML for in-database machine learning
   - Gemini Enterprise for code generation and analysis
   - Looker with LookML semantic layer for BI
   - Vector search capabilities for AI applications

5. **Cost Optimization**
   - Moving from on-demand to flat-rate BigQuery slots
   - Estimated commitment: 500 slots based on initial migration
   - Storage optimization with table partitioning and clustering

**Key Technical Decisions:**
- ELT over ETL (transform in BigQuery for performance)
- Separate non-prod and prod ingestion pipelines
- Governance team owns taxonomy, engineering applies tags
- Use of reservation assignments for workload isolation
